## 1. CLI：单图生成

```bash
python3 scripts/generate_player_radar.py \
  --input <input_csv> \
  --output <output_png> \
  --title "<主标题>" \
  --subtitle "<副标题>" \
  --dpi 220
```

示例：

```bash
python3 scripts/generate_player_radar.py \
  --input templates/player_chart_template.csv \
  --output out/player_charts/alberto_quiles.png \
  --title "Alberto Quiles Piosa (30, CF, 2901 mins.), Tianjin Tigers" \
  --subtitle "2025 Chinese Super League Percentile Rankings & Per 90 Values"
```

参数：

- `--input`：输入 CSV（必填）
- `--output`：输出 PNG（必填）
- `--title`：主标题（必填）
- `--subtitle`：副标题（可选）
- `--dpi`：图片清晰度，默认 `220`

---

## 2. CSV 字段规则

必填字段：

- `metric`
- `value`（0-100）
- `group`
- `order`（整数）

可选字段：

- `subOrder`
- `per90`
- `tier`（`elite/above_avg/avg/bottom`）
- `color`

## 3. Web：启动方式

先进入仓库根目录（本机示例）：

```bash
cd /Users/yangchangran/xiangmu
```

推荐（默认）：一键启动（自动检查后端健康，不健康则自动拉起后端，再启动前端）

```bash
bash scripts/start_player_web_dev.sh
```

启动后可验证后端健康：

```bash
curl -s http://127.0.0.1:8787/api/health
```

备选：双终端手动启动（调试时使用）

```bash
# 1) 后端（终端 A）
cd player-web/server
python3 -m pip install -r requirements.txt
python3 app.py
```

后端启动后，建议至少验证一次健康检查：

```bash
curl -s http://127.0.0.1:8787/api/health
```

```bash
# 2) 前端（终端 B）
cd /Users/yangchangran/xiangmu/player-web
npm install
npm run dev -- --host 127.0.0.1 --port 5173
```

可选环境变量：

- `VITE_STORAGE_API_BASE`：后端地址，默认 `http://127.0.0.1:8787`

生产部署（Ubuntu 24.04 / IP 直连）：

- 生产部署资产位于 `deploy/player-web-prod/`
- 前端构建时使用 `VITE_STORAGE_API_BASE=/`，通过同源 `/api/*` 访问后端
- 后端生产运行使用 `gunicorn` 单进程绑定 `127.0.0.1:8787`
- `Nginx` 对外提供静态页面，并将 `/api/*` 反代到 `127.0.0.1:8787`
- 生产默认对外端口为 `80`；可通过环境变量 `PLAYER_WEB_PUBLIC_PORT` 改为其他端口（如 `8080`）
- 生产默认启用站内共享账号登录页；登录成功后才可进入工作台
- 登录态使用浏览器会话 cookie；关闭浏览器后重新打开，需要再次登录
- 顶部导航提供“退出登录”按钮；点击后当前浏览器会话立即回到登录页
- 生产环境支持多个共享账号；账号列表写入 `/etc/player-web/auth.json`
- 为避免当前 JSON 落盘在多进程下出现并发写风险，生产默认只启 `1` 个 Python worker

服务器安装命令：

```bash
sudo bash deploy/player-web-prod/scripts/install_player_web_prod.sh
```

若不使用 `80/443`，可改为自定义端口：

```bash
sudo PLAYER_WEB_PUBLIC_PORT=8080 \
  bash deploy/player-web-prod/scripts/install_player_web_prod.sh
```

服务器更新命令：

```bash
sudo bash deploy/player-web-prod/scripts/update_player_web_prod.sh
```

如需显式变更对外端口：

```bash
sudo PLAYER_WEB_PUBLIC_PORT=8080 \
  bash deploy/player-web-prod/scripts/update_player_web_prod.sh
```

修改线上共享账号/密码：

```bash
sudo bash deploy/player-web-prod/scripts/set_player_web_shared_login.sh
```

追加一个共享账号：

```bash
sudo bash deploy/player-web-prod/scripts/add_player_web_shared_login_account.sh
```

服务器数据备份命令：

```bash
sudo bash deploy/player-web-prod/scripts/backup_player_web_data.sh
```

Web 技术事实：

- 前端源码位于 `player-web/src`
- 源码统一为 TypeScript（`.ts/.tsx`）
- 样式入口为 `player-web/src/styles.css`，分层文件位于 `player-web/src/styles/*.css`
- 不变更现有后端接口路径与 JSON 字段口径
- 当前结构约定：`App.tsx` 负责应用壳层，雷达图纯逻辑位于 `player-web/src/app/radar/*`，后端入口 `player-web/server/app.py` 仅负责注册蓝图

结构整理不变性：

- 可以拆分页面组件、hooks、utils、routes、services
- 不得改变页面功能、接口路径、JSON 字段、默认值、回退策略和统计口径

## 4. Web 功能行为（按页面）

## 4.1 导航

顶部导航采用“一级菜单 + 下拉菜单”：

- 主页（一级菜单）
- 数据雷达图（下拉：球员数据、雷达图生成器）
- 数据散点图（下拉：数据散点图、个人雷达图）
- 比赛数据总结（下拉：球队数据、比赛雷达图）
- 体能数据分析（下拉：两队体能雷达图、球员体能叠加雷达、分均体能数据）
- opta数据分析（一级菜单）
- 中超积分走势（一级菜单）
- 对应表（下拉：项目对应表、比赛项目对应表、姓名对应表、球队对应表）
- About（一级菜单）

下拉菜单支持两种触发方式：鼠标悬停展开、点击展开/收起。

左上角品牌文案：`生成器V3.3`

## 4.2 球员数据

- 支持导入 Excel（`.xlsx`）
- 必须包含 `player` 列，按宽表读取（一行一个球员）
- 导入前会检查后端健康状态；当 `http://127.0.0.1:8787/api/health` 不可用时，导入按钮禁用并提示
- 数值列自动参与排名与百分比
- 若 Excel 存在重复表头，同名列按“优先保留有数据的值”合并；空重复列不会覆盖前面的有效值
- 支持按数据集切换、删除、搜索球员、选择球员
- 支持按数据集勾选指标，一键导入雷达图生成器
- 支持将“已勾选指标”保存为多个全局预设，并支持在任意数据集下直接选择应用、改名、删除
- 指标预设全局保存到后端状态；每个数据集会单独记住上次选中的预设；选择预设时会直接覆盖当前勾选
- 预设跨数据集应用时按原始 Excel 列名匹配当前数据集可用数值列；缺失列会忽略并提示
- 删除球员数据数据集时，不删除全局指标预设

一键导入口径：

- `value = 百分比`
- `per90 = 该球员原始列值`
- `metric` 优先使用项目对应表中文名
- `group` 优先使用项目对应表配置
- 导入到雷达图后，标题模板中的球员英文名、中文名、年龄、位置会同步更新（中文来自姓名对应表）
- 导入到雷达图后，会自动清除雷达图左上角图片（避免沿用上一个球员照片）

雷达图生成器补充：

- 左侧底部“数据表”支持展开/收起开关（默认展开）
- 图表样式支持设置右图背景颜色（默认 `#f8f5ef`）
- 图表样式区桌面端采用三列排布
- 图片设置区已对齐为统一选项块间距与紧凑行距
- 左上角图片每次上传后默认重置为：`size=200px`、`x=0`、`y=120`（上传后仍可手动调整）
- 导出 PNG 文件名规则为“版本名 + 两位序号”；已选版本使用当前版本名，草稿状态优先使用输入框中的版本名，兜底为“当前草稿”

主页入口补充：

- 主页聚合展示全部导航功能入口（含 About），支持卡片直达对应页面。
- 主页仅展示一级导航入口，不展示下拉子菜单链接。
- 主页功能卡片不显示“一级菜单”字样。
- “对应表”入口在主页仅展示，不触发跳转。
- 主页不展示操作方法步骤区，仅保留视觉化入口与核心信息卡。
- 主页“全部功能入口”上方不再展示三项说明框。

## 4.3 散点图生成器

基础能力：

- 选择 X/Y 轴
- 可在“数据集控制”里删除当前数据集
- 按范围筛选点
- 悬浮提示、点选联动详情
- 显示球员名字开关
- 球员名字字号可调（数字输入 + 滑块）
- 图表背景颜色可调
- 点大小、边框颜色、边框粗细可调
- 平均线开关、平均线颜色、平均线粗细可调

关键口径：

- 平均值计算基于“全量有效点（X/Y 可解析）”
- 改 `X/Y 最小值最大值` 只影响可见点与坐标缩放，不改变平均值统计口径

球队样式联动：

- 按“球队对应表” `English` 字段匹配球队（大小写不敏感）
- 匹配成功：使用对应 `color` 与 `shape`
- 未匹配：颜色回退为黑色
- 数据集无球队列（Team/Club/Squad）：保持默认样式并提示
- 球员名字显示优先使用“姓名对应表”中文名，未命中回退英文名

雷达标题联动：

- 标题中的球员名使用 `English 中文`（中英文之间空格分隔，中文来自姓名对应表）
- 未命中中文时仅显示英文名
- 标题模板支持手动填写“中文名”输入框；填写后优先使用手填中文

## 4.4 项目对应表

- 维护 Excel 列的中英映射
- 支持维护 `group`
- 支持下载对应表 CSV / Excel，并可用同一份 CSV / Excel 直接回放导入
- CSV 表头固定为 `English,中文翻译,group`
- Excel 表头固定为 `English,中文翻译,group`
- CSV / Excel 导入会以文件内容整体覆盖当前项目对应表；内置项目若未出现在导入文件中，则会在页面中隐藏
- 兼容旧的“任意 Excel 首行表头导入”，未按对应表模板填写时仍会按首行列名生成项目
- 支持手动新增项目
- 支持删除项目：内置已有项目执行“仅页面隐藏”，不改变全局映射；新增项目执行实际删除
- 内容保存到后端共享状态，刷新页面、切换浏览器或重启后端后仍保留
- 云端已有数据时，以后端共享状态为准；首次迁移时会把本机浏览器中的历史表内容写入后端

## 4.5 比赛项目对应表

- 仅用于比赛总结（球队数据/比赛雷达图）的中英文映射与分组维护
- 支持下载对应表 CSV / Excel，并可用同一份 CSV / Excel 直接回放导入
- CSV 表头固定为 `English,中文翻译,group`
- Excel 表头固定为 `English,中文翻译,group`
- CSV / Excel 导入会以文件内容整体覆盖当前比赛项目对应表
- 兼容旧的比赛总结双表头 Excel 解析：若导入文件不是对应表模板，仍会按比赛总结表头拆分生成项目
- 支持手动新增、删除
- 与“项目对应表”（球员数据域）完全隔离
- 内容保存到后端共享状态，三人访问同一网站时共享同一份比赛项目对应表

## 4.6 姓名对应表

- 维护球员英文名/中文名/球队
- 支持下载姓名对应表 CSV / Excel，并可用同一份 CSV / Excel 直接回放导入
- CSV 表头固定为 `English,中文翻译,球队`
- Excel 表头固定为 `English,中文翻译,球队`
- CSV / Excel 导入会以文件内容整体覆盖当前姓名对应表
- 兼容旧的球员数据 Excel 导入：若导入文件不是对应表模板，仍会按 `Player/Name` 与球队列提取姓名
- 支持“从球员数据同步姓名”（增量合并）
- 中文翻译默认“保留已有中文，仅补空值”，并支持离线音译自动补全
- 支持“批量补全中文名”（仅填空，不覆盖）
- 支持按“球队”列排序（仅调整行顺序，不修改单元格内容）
- 支持一键删除现有姓名（清空姓名对应表）
- 内容保存到后端共享状态，刷新页面、换浏览器或服务器重启后仍保留

## 4.7 球队对应表

- 维护球队英文名/中文名
- 维护 `color` 与 `shape`
- 支持下载球队对应表 CSV / Excel，并可用同一份 CSV / Excel 直接回放导入
- CSV 表头固定为 `English,中文翻译,color,shape,logoFileName`
- Excel 表头固定为 `English,中文翻译,color,shape,logoFileName`
- CSV / Excel 导入会以文件内容整体覆盖当前球队对应表
- CSV 不包含 Logo 图片本体；`logoFileName` 仅用于记录文件名，真正图片仍通过单队上传或批量导入 Logo 维护
- Excel 同样不包含 Logo 图片本体；导入时只恢复 `logoFileName` 文本列，现有 Logo 图片继续按页面上传结果保留
- 支持单队上传 logo，并保存到后端共享状态
- 支持批量导入 logo（按文件名匹配球队中/英文）
- 支持颜色与形状示意
- 支持“从球员数据同步球队”（增量合并，不覆盖已填中文）
- 内容保存到后端共享状态，三人访问同一网站时共享同一份球队映射与 logo

## 4.8 比赛总结 / 球队数据

- 独立于球员数据域，使用独立后端接口 `/api/match-data/*`
- 支持导入 Excel（`.xlsx`），必须包含 `Team` 列
- 支持数据集切换、删除、搜索球队、选择主队/客队
- 指标列表为主客队共同可解析数值列（交集）
- 支持将“已勾选指标”保存为多个全局球队指标预设，并支持在任意比赛数据集下直接选择应用、改名、删除
- 球队指标预设全局保存到后端状态；每个比赛数据集会单独记住上次选中的预设；选择预设时会直接覆盖当前勾选
- 球队指标预设跨数据集应用时按原始 Excel 列名匹配当前数据集可用数值列；缺失列会忽略并提示
- 删除比赛数据数据集时，不删除全局球队指标预设
- 支持勾选指标并一键导入“比赛雷达图”（导入原始值，不导入排名/百分比）
- 导入到比赛雷达图时，会优先匹配 `Goals/进球/进球数` 列，并把主客队该列值自动写入比赛雷达图比分
- 导入只作用于“比赛雷达图”，不影响球员数据、散点图、雷达图生成器
- 列标题显示与分组优先使用“比赛项目对应表”

## 4.9 比赛总结 / 比赛雷达图

- 使用同一比赛数据集内“主队/客队”双队对比
- 使用主客队原始值绘图，不使用排名/百分位
- 支持编辑主标题/副标题、主客队比分、主客队颜色
- 从“球队数据”一键导入时，若命中 `Goals/进球` 类列，会自动覆盖主客队比分；未命中时保留当前手填比分
- 副标题默认优先使用“球队数据”表格中的比赛日期（优先匹配日期列；若未命中则回退到首列日期值；仍未识别时回退为“数据来源：比赛总结 / 球队数据”）
- 支持主客队 logo 下拉选择（来源：球队对应表，支持自动匹配）
- 支持比分字号调节（36-120，滑块+数字输入）
- 支持调节点大小（2-10，统一作用于主客队两条雷达折线的节点）
- 支持设置右图背景颜色（位于“导出 SVG”前，默认 `#f8f5ef`）
- `PPDA` 指标采用反向映射（值越小越靠外，值越大越靠内），仅影响右图点位，不改左侧表格与标签显示原值
- 支持按指标调节“在雷达图中的顺位位置”（左下数据表“位置”列数字输入，向上/向下移动该指标在环上的位置）
- 每个指标支持手动设置 `max`（`min` 固定为 0）
- 指标未手动设置 `max` 时，默认取 `max(主值, 客值) * 1.1`
- 支持导出 SVG/PNG
- 与原“雷达图生成器”状态和草稿完全隔离

## 4.10 体能数据分析

- 支持导入 Excel（`.xlsx`），按 Sheet 名称口径解析（不再依赖“前 2 个 Sheet”顺序）
- 使用独立后端接口 `/api/fitness-data/*` 与独立数据集管理
- 导航拆分为三个子页面：
  - 两队体能雷达图（第一页）
  - 球员体能叠加雷达（第二页）
  - 分均体能数据（第三页）
- Sheet1（第1个 Sheet）用于“两队体能雷达图”：
  - 优先使用名称为 `Overview` 的 Sheet 作为两队体能雷达图数据源
  - 自动识别球队数据结构：
    - 标准表格式：识别球队列（优先 `Team`，未命中回退 `team/club/squad/球队/俱乐部`）
    - Overview 模板：识别 `主队/数据/客队`（或 `home/data/away`）布局，并自动定位对应数值列
  - 只取前 2 支有效球队并生成对比
  - 上传入口位于第一页；上传后第二页自动同步到同一份新数据集
  - 下方数据表按“全部可用指标”展示；每行指标前内置勾选框，控制是否参与雷达图
  - 指标默认全选；雷达值使用两队原始值，不做排名/百分位换算
  - 每行支持手动输入该指标雷达 `max`；为空时默认 `max(主值, 客值) * 1.1`
  - 手动 `max` 按“数据集 + 指标”持久化保存；“清空勾选”仅影响勾选状态，不重置手动 `max`
  - 支持主标题/副标题、队伍颜色、点大小、背景色调节，支持导出 SVG/PNG
  - 副标题不因每次导入自动重置，保持用户当前设置
  - 球队颜色默认按“球队对应表”自动映射，未命中时回退默认颜色
  - 每次导入体能 Excel 后，当前新数据集主客队颜色都会按“球队对应表” `color` 强制同步（支持中英文队名匹配）
  - 当导入队名已是中文时，标题显示不再用“球队对应表”的中文名覆盖，直接保留导入值
  - 主客队标题上方显示队徽（来源：球队对应表 `logo`），未命中时回退字母占位

## 4.13 中超积分走势

- 导航新增一级菜单：`中超积分走势`
- 使用独立后端接口 `/api/csl-standings/*` 与独立数据集管理
- 支持导入赛程 Excel（`.xlsx`），默认读取活动 Sheet
- 统计口径：
  - 仅统计 `联赛名称` 命中“中超”的比赛
  - 仅统计 `比赛状态=比赛结束` 的已完赛比赛
  - 提供双口径榜单与走势：`原积分/原排名` 与 `净积分/净排名（含扣分）`
  - 扣分规则仅在 `2026` 赛季生效；非 2026 赛季扣分恒为 `0`
  - 排名规则（原/净口径一致）：`积分 > 净胜球 > 进球 > 球队名`，名次连续且不重复
  - 排名走势图展示层会按同一口径做唯一名次校正，避免同轮出现重复排名
  - 补赛按“轮次归属”回填到对应轮次（不按实际比赛时间后置）
- 支持赛季切换（联赛固定中超）
- 页面展示：
  - 左侧：每轮积分榜（净排名/原排名/扣分/净积分/原积分/进球/丢球/净胜球/场次）
  - 右侧：多队同图走势，支持勾选指标：`原积分/净积分/原排名/净排名/进球/丢球`
  - 新增“动态播放”控制：可从 Round 0 逐轮播放到最后一轮，折线按轮次递进揭示
  - 动态播放采用轮间平滑插值（缓动），仅改变过渡观感，不改变各轮离散统计值
  - 动态播放时“当前运动点”可显示球队 logo（来源：球队对应表 `logoDataUrl`）
  - 动态播放结束后，logo 默认停留在结束轮次点位继续显示
  - logo 缺失或加载失败时，运动点回退为球队颜色圆点
  - `净积分（pointsNet）` 走势图坐标口径：纵轴下界固定 `-10`，横向主坐标轴固定在 `0` 分位置
  - 球队颜色优先按“球队对应表” `color` 映射（支持中英文队名匹配）
  - 球队未命中或颜色非法时：回退中超积分走势内置调色板
  - 点击左侧球队名称或图内趋势线可锁定该队趋势线高亮（再次点击取消）；该操作不改变复选框勾选状态
  - 存在锁定高亮时，点击图内空白区域可取消锁定高亮
  - 同轮同值（如积分/进球/丢球相同）点位重叠时：点位在显示层左右并排展开；折线仍使用真实坐标，不改变统计口径
  - 高亮时趋势线加粗，非高亮趋势线降低透明度，以强化高亮对比
  - 悬浮高亮时同时显示竖向与横向虚线，右上角展示该轮焦点球队指标值
- 默认行为：
  - 默认显示全部球队
  - 默认勾选六个指标（原积分/净积分/原排名/净排名/进球/丢球）
  - 默认轮次为“当前赛季最新已完赛轮次”（积分榜轮次列表含 Round 0 基线）
  - 动态播放默认“手动触发”，不自动播放
  - 点击“播放”时默认从 `Round 0` 开始逐轮播放
  - 播放到最后一轮后自动暂停并停在末轮
  - 播放速度支持滑条调节：`200-2000ms/轮`，步进 `100ms`，默认 `800ms/轮`
  - 手动切换左侧轮次会暂停播放，并将动态播放轮次同步到手动所选轮次
  - 图表高度默认 `340px`；纵轴步长固定为：`排名/净排名=2`，`积分/净积分/进球/丢球=5`
- 导入失败时返回明确错误（缺列、无有效中超完赛数据、文件格式错误）

## 8. PDF 中文覆盖西语（离线）

用于在不改变底图结构（图片/表格线）的前提下，把 PDF 中文文本块覆盖为西语文本。

```bash
python3 scripts/pdf_overlay_es.py \
  <input.pdf> \
  <output.pdf> \
  --mapping templates/pdf_overlay_es_mapping.json
```

参数：

- `input.pdf`：输入 PDF（必填）
- `output.pdf`：输出 PDF（必填）
- `--mapping`：翻译映射 JSON，默认 `templates/pdf_overlay_es_mapping.json`

映射文件结构：

- `exact`：整段精确替换（`cn` -> `es`）
- `phrase`：短词/表头子串替换

- `out/architecture_audit_report.json`
- `out/architecture_audit_report.md`

任一规则违规，命令返回非零退出码。

若为“功能不变”的结构重排，交付中还必须说明：

- 哪些入口文件被拆分
- 新的职责分别落在什么目录
- 哪些行为被明确验证为不变
