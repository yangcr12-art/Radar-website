# Roadmap (player)

> 执行顺序：P0（可用）→ P1（高效）→ P2（产品化）

---

## P0（当前必须完成）

目标：把“CSV 填数值 -> 自动出图”做成稳定工具。

1. 模板稳定化
- 固定 CSV schema（必填/可选字段）
- 增加字段校验与报错定位（行号级）

2. 出图一致性
- 统一默认配色（tier 映射）
- 统一默认布局（分组、内环、标签）

3. 基础可复现
- 文档给出一键命令
- 所有示例命令可直接跑

DoD：
- [ ] 新用户按 `README + USAGE` 可在 5 分钟内出第一张图
- [ ] 报错信息能定位到具体字段/行

---

## P1（效率提升）

目标：支持批量生产和团队协作。

1. 批量导图
- 新增批处理脚本（读取目录内多个 CSV）
- 输出命名规则统一

2. 主题系统
- 俱乐部主题色配置（json/yaml）
- 深浅背景方案切换

3. 输出元信息
- 每次导图生成 metadata（输入文件、时间、参数）

DoD：
- [ ] 一条命令导出 N 名球员图
- [ ] 主题切换不改代码，只改配置

---

## P2（产品化）

目标：让非技术用户也能使用。

1. Web UI
- 上传 CSV
- 在线预览
- 一键导出 PNG

2. 模板管理
- 前锋/中场/后卫模板预设
- 自定义模板保存

3. 数据质量报告
- 自动识别异常指标
- 导出校验报告

DoD：
- [ ] 无需命令行即可完成导图
- [ ] 校验报告可用于审核流程

---

## 非目标（当前阶段）

- 不在本项目内实现完整爬虫/数据采集平台
- 不做复杂机器学习模型训练
- 不做实时比赛分析系统
