# Roadmap (player)

执行顺序：`P0（稳定与审计） -> P1（效率与体验） -> P2（产品化与扩展）`

---

## 1) 当前状态（截至 V3.3）

已完成：

- 雷达图模板化出图主链路
- 球员数据页 Excel 导入与排名百分比
- 散点图生成器（X/Y 选择、筛选、详情联动、样式控制）
- 项目对应表与球队对应表
- 散点图按球队映射颜色/形状
- 文档体系升级为规则化协作
- Web 前端源码统一迁移为 TypeScript（保持行为口径兼容）

待加强：

- 文档与代码自动一致性校验
- Web 端回归场景脚本化
- 交互复杂度增长后的状态治理

---

## 2) P0（必须完成）

目标：稳定口径与审计闭环。

1. 规则稳定化
- 将关键默认值集中声明并做文档索引
- 增强缺列/未匹配/空值回退提示一致性

2. 审计闭环
- 文档变更触发条件落实到任务模板
- 每次交付输出“规则变更摘要”

3. 回归基线
- 固化最小验证命令与结果格式
- 增加 Web build 作为默认验证项（前端变更时）

DoD：

- [ ] 默认值、回退策略在 `USAGE.md` 可直接查到
- [ ] 任意功能改动都能明确是否影响口径

---

## 3) P1（效率提升）

目标：减少重复操作，提升产出效率。

1. 批量导图增强
- 支持目录批处理与命名策略

2. 主题系统
- 颜色、字体、背景方案配置化

3. 数据到图表链路优化
- 一键导入与映射配置可视化提示更明确

4. QA 支持能力
- 增加页面级回归检查清单（手工/半自动）

DoD：

- [ ] 一条命令/流程可批量导出多个球员图
- [ ] 主题切换不改代码，仅改配置

---

## 4) P2（产品化）

目标：让非技术用户可稳定使用。

1. Web 交互增强
- 模板预设、页面导引、错误自诊断

2. 报表体系
- 导出“数据质量报告 + 规则说明”

3. 协作能力
- 版本快照与对比
- 可追溯变更说明自动生成

DoD：

- [ ] 无命令行即可完成导入、分析、导出
- [ ] 可对关键口径变更进行可视化追踪

---

## 5) 非目标（当前阶段）

- 不扩展为全链路数据抓取平台
- 不引入复杂机器学习训练系统
- 不做实时赛事级流处理

---

## 6) 经验教训（路线图层）

1. 先固规则再堆功能，能显著减少返工。
2. UI 需求增长时，最先要做的是状态边界梳理，而不是继续叠控件。
3. 每次新增默认行为都应视为“产品规则变更”，必须文档化。
